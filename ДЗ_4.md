## Задача 3

**Подключение к потокам контейнера:**
```bash
docker attach custom-nginx-t2
```
После подключения нажата комбинация Ctrl-C.

**Почему контейнер остановился:**

`docker attach` подключается к основному процессу контейнера (PID 1 — nginx). Ctrl-C отправляет сигнал `SIGINT` этому процессу, nginx завершается, и контейнер останавливается вместе с ним.

```bash
docker ps -a
docker start custom-nginx-t2
docker exec -it custom-nginx-t2 bash
```

Внутри контейнера:
```bash
apt-get update && apt-get install -y nano
nano /etc/nginx/conf.d/default.conf  # заменить listen 80 на listen 81
nginx -s reload
curl http://127.0.0.1:80
curl http://127.0.0.1:81
exit
```

**Проверка снаружи:**
```bash
ss -tlpn | grep 127.0.0.1:8080
docker port custom-nginx-t2
curl http://127.0.0.1:8080
```

**Суть возникшей проблемы:**

При запуске контейнера был проброшен порт `8080 → 80`. После изменения конфига nginx стал слушать порт `81`, а не `80`. Docker проксирует трафик с хостового `8080` на контейнерный `80`, но там уже никто не слушает — отсюда ошибка `Connection reset by peer`. Маппинг портов задаётся при старте контейнера и не меняется на лету.

**Удаление контейнера без остановки:**
```bash
docker rm -f custom-nginx-t2
```

## Задача 5

**Создание директории и файлов:**
```bash
mkdir -p /tmp/netology/docker/task5 && cd /tmp/netology/docker/task5
```

**compose.yaml:**
```yaml
version: "3"
include:
  - docker-compose.yaml
services:
  portainer:
    network_mode: host
    image: portainer/portainer-ce:latest
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
```

**docker-compose.yaml:**
```yaml
version: "3"
services:
  registry:
    image: registry:2
    ports:
    - "5000:5000"
```

**Какой файл запустился и почему:**

При первом запуске `docker compose up -d` запустился только `compose.yaml`. По спецификации Docker Compose, при наличии обоих файлов (`compose.yaml` и `docker-compose.yaml`) приоритет имеет `compose.yaml` (без дефиса). Файл `docker-compose.yaml` игнорируется.

**Запуск обоих файлов через include:**

В `compose.yaml` добавлена директива `include` которая подключает `docker-compose.yaml`. После этого `docker compose up -d` запускает оба сервиса.

**Загрузка образа в локальный registry:**
```bash
docker pull zolotkoff/custom-nginx:1.0.0
docker tag zolotkoff/custom-nginx:1.0.0 127.0.0.1:5000/custom-nginx:latest
docker push 127.0.0.1:5000/custom-nginx:latest
```

**Деплой стека в Portainer:**
```yaml
version: '3'

services:
  nginx:
    image: 127.0.0.1:5000/custom-nginx
    ports:
      - "9090:80"
```

**Удаление compose.yaml и предупреждение:**
```bash
rm compose.yaml
docker compose up -d
```

**Суть предупреждения:**

Docker Compose обнаружил контейнеры (`portainer`), которые были запущены ранее из `compose.yaml`, но теперь этот файл удалён и они не описаны ни в каком активном файле — они стали «осиротевшими» (orphan containers). Compose предлагает удалить их командой `--remove-orphans`.

**Погасить весь проект одной командой:**
```bash
docker compose down --remove-orphans
```
