## Задача 3

1. Изучен файл `proxy.yaml` (haproxy на `172.20.0.5:5000`, nginx на host network, порт 8090).

2. Создан `compose.yaml` с подключением `proxy.yaml` через `include`:

```yaml
include:
  - proxy.yaml

services:
  web:
    build:
      context: .
      dockerfile: Dockerfile.python
    restart: always
    networks:
      backend:
        ipv4_address: 172.20.0.5
    environment:
      DB_HOST: db
      DB_USER: ${MYSQL_USER}
      DB_PASSWORD: ${MYSQL_PASSWORD}
      DB_NAME: ${MYSQL_DATABASE}
    env_file:
      - .env

  db:
    image: mysql:8.0
    volumes:
      - ./mysql-init:/docker-entrypoint-initdb.d
    restart: always
    networks:
      backend:
        ipv4_address: 172.20.0.10
    env_file:
      - .env
```

> **Примечание:** зафиксирована версия `mysql:8.0` вместо плавающего тега `mysql:8` (который сейчас указывает на 8.4.x). Это понадобилось для Задачи 5 — образ `schnitzler/mysqldump` использует устаревший клиент без поддержки `caching_sha2_password`, полностью убранного в MySQL 8.4. Версия 8.0 остаётся в рамках требуемой линейки `mysql:8`.

## Задача 5 (*)

1. Создана директория `/opt/backup`.

2. Написан `backup.sh`:

```bash
#!/bin/bash
set -e

# Загружаем переменные окружения из .env (не светим пароль в коде!)
source /opt/shvirtd-example-python/.env

BACKUP_DIR="/opt/backup"
TIMESTAMP=$(date +"%Y%m%d_%H%M%S")

docker run --rm \
  --entrypoint "" \
  --network shvirtd-example-python_backend \
  -v "${BACKUP_DIR}:/backup" \
  schnitzler/mysqldump \
  mysqldump --opt -h db -u root -p"${MYSQL_ROOT_PASSWORD}" --result-file="/backup/dump_${TIMESTAMP}.sql" "${MYSQL_DATABASE}"

echo "Backup completed: ${BACKUP_DIR}/dump_${TIMESTAMP}.sql"
```

> **Техническая проблема и решение:** образ `schnitzler/mysqldump` использует старый MySQL-клиент без поддержки `caching_sha2_password` (дефолтный плагин аутентификации в MySQL 8+). Решено через init-скрипт `mysql-init/01-auth.sh`, который при первом старте контейнера `db` переключает root на совместимый `mysql_native_password`:

```bash
#!/bin/bash
set -e
mysql -u root -p"${MYSQL_ROOT_PASSWORD}" -e "ALTER USER 'root'@'%' IDENTIFIED WITH mysql_native_password BY '${MYSQL_ROOT_PASSWORD}'; FLUSH PRIVILEGES;"
```

Скрипт подключён через volume в `compose.yaml`:
```yaml
volumes:
  - ./mysql-init:/docker-entrypoint-initdb.d
```

3. Ручной запуск протестирован:
```bash
sudo bash backup.sh
```
Дамп успешно создан (`mysqldump 10.19-MariaDB` дамп базы `virtd`).
